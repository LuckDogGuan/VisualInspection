# 铝型材现场检测与缺陷区域判断计划

更新时间：2026-06-22

## 1. 当前结论

当前项目先按低成本、可验证、可逐步升级的方式推进。

短期目标不是直接做全自动剔除，而是先做辅助人工判断：

```text
固定相机拍照
模型判断是否疑似缺陷
疑似缺陷时保存原图和结果图
结果图显示缺陷类别、置信度和大致位置
工人根据屏幕结果决定是否拿走材料
```

后续真正需要框选缺陷位置时，优先走检测模型路线，而不是只依赖分类模型。

## 2. 现场拍摄建议

建议现场按以下原则布置：

```text
相机：固定在流水线上方
拍摄方向：尽量垂直向下
光源：固定条形光、环形光或面光
背景：单一、哑光、减少反光
触发：优先使用光电传感器或编码器触发拍照
位置：让铝型材每次经过画面中尽量固定的位置
```

最重要的是稳定，而不是一开始追求复杂设备。

必须尽量稳定：

```text
拍摄角度
拍摄距离
曝光参数
光源亮度
背景颜色
材料在画面中的位置
```

如果现场反光明显，建议增加遮光罩或固定光源角度。反光会明显影响分类和后续框选效果。

## 3. 流水线显示方案

第一版不做复杂追踪，只显示最近一次或最近几次检测结果。

推荐流程：

```text
材料经过相机
触发拍照
模型推理
正常：显示绿色正常
疑似缺陷：显示红色异常、缺陷类别、置信度、结果图
保存异常原图和结果图
工人根据屏幕提示人工决定是否拿走
```

界面建议：

```text
左侧：当前相机原图
右侧：带标注的结果图
底部：最近 20 条检测记录
```

每条记录包含：

```text
检测时间
图片编号
判断结果
缺陷类型
置信度
保存路径
```

如果后续需要把屏幕提示和流水线物理位置对应起来，再增加传感器延时、编码器位置或 PLC 联动。

## 4. 缺陷区域判断路线

### 阶段一：分类模型结果图

先使用当前分类模型，生成带标题的结果图：

```text
正常
疑似缺陷
预测类别
置信度
Top3 候选
```

这一阶段不要求精确框选，只用于快速展示和保存结果。

### 阶段二：粗定位热力图

可以使用 Grad-CAM 这类热力图方法，显示分类模型大概关注的位置。

优点：

```text
不需要人工框选标注
可以基于当前分类模型尝试
成本低
```

限制：

```text
位置不一定准确
只能作为参考
不能作为正式检测框
```

阶段二已完成第一版代码和样例运行。

实现脚本：

```text
src/classification/generate_gradcam.py
```

运行方式：

```bash
python src/classification/generate_gradcam.py \
  --checkpoint outputs/classification_results/exp_focal/models/model_best.pth.tar \
  --output-dir outputs/classification_results/gradcam_exp_focal_samples \
  --per-class 3
```

服务器运行结果：

```text
输出图片数：33
服务器输出目录：outputs/classification_results/gradcam_exp_focal_samples
本地下载目录：server_outputs/gradcam_exp_focal_samples
报告文件：gradcam_report.csv
说明文件：summary.md
```

当前结论：

```text
Grad-CAM 可以作为粗定位参考。
它能显示分类模型大概关注的区域。
它不能替代 YOLO 检测框。
如果热力图经常落在缺陷附近，说明分类模型关注点有价值。
如果热力图经常落在背景、边缘、反光区域，说明需要改进拍摄条件或训练数据。
```

下一步建议：

```text
先人工查看本次 33 张热力图样例。
挑出热力图有效和无效的例子。
如果大多数热力图对人工有帮助，再把 Grad-CAM 加到现场演示结果图中。
如果热力图不稳定，不继续深挖，直接进入 YOLO 小样本框选验证。
```

### 阶段三：YOLO 检测模型

后续正式框选缺陷区域，推荐训练 YOLO 检测模型。

检测模型输出：

```text
缺陷类别
缺陷框位置
置信度
```

检测模型更适合处理一张图中多个缺陷的情况，例如：

```text
框 1：擦花
框 2：脏点
框 3：碰伤
```

阶段三不要直接做全量训练，先做一个小样本验证闭环。目标是验证图片质量、标注规则、缺陷类别定义和 YOLO 是否能框出大致位置，而不是第一版就追求完整生产效果。

#### 阶段三第一批类别选择

第一批优先选择容易看清、边界相对明确、现场价值高的类别：

```text
dent：碰伤
powder：凸粉
stain：脏点
crack：涂层开裂
transverse_bump：横条压凸/横条压凹类
```

暂时不优先标注：

```text
不导电
擦花
桔皮
漏底
轻微浅色缺陷
反光区域缺陷
```

原因：

```text
不导电通常不是一个清晰局部框，容易变成整片区域判断
擦花有时又细又长，边界不稳定，早期容易造成标注标准不统一
桔皮偏纹理类缺陷，框选边界主观性强
轻微浅色缺陷和反光区域缺陷容易把光照问题误当缺陷
```

这些类别后续可以做第二批，等第一批 YOLO 能稳定框出明显缺陷后再加入。

#### 阶段三挑图规则

图片已经按标准类别分类时，先按目录类别挑图，再人工筛选清晰度和典型性。不要只按数量硬凑。

每一类先挑：

```text
第一轮：每类 10 到 20 张，用于确认标注标准
第二轮：每类补到 30 到 50 张，用于第一次训练
总量：先控制在 200 到 500 张
```

优先选择：

```text
单一缺陷类型
缺陷位置看得清楚
缺陷区域和正常区域有明显差异
图片没有水印、截图边框、人工文字、旧标注框
图片不模糊、不严重过曝、不严重反光
同一角度、同一背景、同一光照条件下的图片优先
```

暂时放入“不确定”：

```text
缺陷看不清
缺陷类型拿不准
疑似反光造成的异常
缺陷太小，缩放后难以判断
一张图里混合多个类别但无法拆开判断
```

直接排除：

```text
重复图片
明显坏图
已经带标注框的结果图
带水印、截图界面、文件名遮挡的图片
不是铝型材表面的图片
```

如果后续有完整分类预测报告，可以叠加置信度筛选：

```text
优先选择预测类别等于目录类别的图片
优先选择 Top1 置信度高的图片
早期建议阈值先取 0.80 以上
如果某类图片不足，再降到 0.70，但必须人工复核
冒烟测试报告不能作为高置信度挑图依据
```

#### 阶段三目录和文件用途说明

当前阶段三目录以 `data/yolo_stage3_manual/` 为准。这个目录的核心原则是：图片用于本地标注和训练流转，标注文件、清单、脚本和文档用于追踪与复现。

```text
data/yolo_stage3_manual/
  README.md
  annotation_registry.csv
  selection_manifest.csv
  candidate_images/
  labelimg_work/
  batches/
  dataset/
  exports/
  review/
  ultralytics_config/
```

各文件和目录作用如下：

```text
README.md
```

阶段三人工标注工作区说明。后续换人继续标注时，先读这个文件，确认类别顺序、当前批次、下一轮标注目标、导出和服务器训练命令。

```text
annotation_registry.csv
```

阶段三最重要的追踪表。每一张进入标注流程的图片都要登记在这里，字段包括批次、状态、类别、源图片路径、批次图片路径、标签路径、挑选时间、标注完成时间。它用于判断“这张图是否已经标注过”，避免重复标注。这个文件需要保存到 Git。

```text
selection_manifest.csv
```

第一轮旧挑图清单，记录最早从分类结果里挑出的图片。后续新批次不再只依赖这个总表，而是每个 `batches/batch_xxx/selection_manifest.csv` 单独记录。这个文件保留用于追溯第一批数据来源。

```text
candidate_images/
```

候选图片暂存目录，用于早期把分类模型筛出的图片按类别临时归档。它不是最终标注入口，也不是训练集。后续主要使用 `batches/batch_xxx/images/` 进行标注。

```text
labelimg_work/
```

第一批标注遗留工作区。`batch_001` 的图片和标签最早放在这里：

```text
labelimg_work/images/
labelimg_work/labels/
labelimg_work/classes.txt
labelimg_work/predefined_classes.txt
```

后续不建议继续把新图片放进这个目录，避免新旧批次混在一起。新标注统一放到 `batches/batch_xxx/`。

```text
batches/
```

每一轮人工标注的独立目录。推荐从这里继续标注：

```text
batches/batch_004/
  images/
  labels/
  classes.txt
  predefined_classes.txt
  selection_manifest.csv
```

`images/` 是本轮要人工标注的图片。`labels/` 是 LabelImg 保存 YOLO `.txt` 标签的位置。`classes.txt` 和 `predefined_classes.txt` 固定类别顺序，不能随意改。`selection_manifest.csv` 是本批次挑图清单，用于追溯每张图来自哪里。

```text
dataset/
```

早期训练数据集目录，保留用于对照。后续正式训练建议使用 `exports/` 下的干净导出目录，而不是直接使用这个目录。

```text
exports/
```

训练前由脚本生成的干净 YOLO 数据集目录。例如：

```text
exports/dataset_annotated_150/
  data.yaml
  export_manifest.csv
  images/train/
  images/val/
  labels/train/
  labels/val/
```

这里只导出 `annotation_registry.csv` 中状态为 `annotated` 的图片和标签。`export_manifest.csv` 用于把导出的英文文件名追溯回原始图片。

```text
review/
```

标注复核材料目录。用于保存标注拼图、第一批复核记录、类别编号修正前的备份等。它服务于检查标注质量，不作为 LabelImg 的工作目录。

```text
ultralytics_config/
```

YOLO/Ultralytics 本地配置目录。它用于减少训练脚本对用户全局配置目录的依赖，属于环境辅助文件，不是标注数据本身。

阶段三脚本用途：

```text
scripts/prepare_yolo_annotation_batch.py
```

从分类评估报告中按每类相同数量挑图，适合第一轮均衡采样。它会复制图片到新 batch，并更新 `annotation_registry.csv`。

```text
scripts/prepare_yolo_priority_batch.py
```

第二轮开始使用的优先级挑图脚本。它可以指定某一类或多类的数量，例如优先准备 `stain=60`，并自动避开已经进入过标注流程的图片。当前分类模型对脏点、凸粉、碰伤的全量筛查置信度不可靠，所以该脚本以“标准分类目录”为主，以分类报告置信度作为附加记录。

```text
scripts/finalize_yolo_annotation_batch.py
```

人工标注完成后运行。它检查本批次标签格式是否正确，把标签文件统一保存为 UTF-8，并把本批次在 `annotation_registry.csv` 中的状态从 `selected` 更新为 `annotated`。

```text
scripts/build_yolo_training_dataset.py
```

把所有已经标注完成的批次导出为干净 YOLO 训练集。只导出 `annotation_registry.csv` 中 `status=annotated` 的记录，未标注完成的 batch 不会进入训练。

```text
scripts/train_yolo_stage3.py
```

服务器训练入口脚本。它读取导出的 `data.yaml`，运行 YOLO 训练，并输出 `results.csv`、`results.png`、预测图和 `training_report.md`。

Git 保存规则：

```text
需要保存到 Git：
- annotation_registry.csv
- selection_manifest.csv
- batches/batch_xxx/selection_manifest.csv
- batches/batch_xxx/labels/*.txt
- batches/batch_xxx/classes.txt
- batches/batch_xxx/predefined_classes.txt
- scripts/*.py
- doc/*.md

不要保存到 Git：
- batches/batch_xxx/images/*
- labelimg_work/images/*
- candidate_images/*
- exports/*/images/*
```

原因是图片体积大、可由源数据和清单追溯，放入 Git 会让仓库膨胀；而标注 `.txt`、清单和登记表是训练复现的关键，必须保存。

`classes.txt` 第一版建议固定为：

```text
dent
powder
stain
crack
transverse_bump
```

顺序一旦开始标注就不要随意改。YOLO 标签里的类别编号依赖这个顺序。

#### LabelImg 标注流程

安装或打开 LabelImg 后，按下面流程操作：

```text
1. 先把要标注的图片复制到 data/yolo_stage3_manual/labelimg_work/images/
2. 打开 LabelImg
3. 点击 Open Dir，选择 labelimg_work/images
4. 点击 Change Save Dir，选择 labelimg_work/labels
5. 确认保存格式为 YOLO
6. 如果界面显示 PascalVOC，点击左侧格式按钮切换到 YOLO
7. 确认 classes.txt 内容和第一版类别表一致
8. 逐张图片画框并选择类别
9. 每张图标完后按 Ctrl+S 保存
10. 按 D 下一张，按 A 上一张
```

标注时的框选规则：

```text
框住缺陷的大致可见区域即可
不要追求像素级贴边
框不要过大，尽量不要包含大量正常背景
一个独立缺陷标一个框
一张图多个缺陷就标多个框
连续的一条缺陷可以用一个长框
密集小点连成一片时可以用一个区域框
看不清、不确定、不知道类别的图片不要强行标，移到 unsure
坏图不要标，移到 bad
反光不是缺陷，除非能确认反光下确实有真实缺陷
```

LabelImg 保存后，每张图片应对应一个同名 `.txt` 文件。例如：

```text
images/碰伤001.jpg
labels/碰伤001.txt
```

YOLO 标签内容格式为：

```text
class_id center_x center_y width height
```

这些数值是归一化坐标，不需要手工填写，由 LabelImg 自动生成。

#### 第一批标注复核流程

不要一次性标完几百张。第一批按下面节奏：

```text
1. 每类先标 5 到 8 张，总数约 30 张
2. 暂停标注
3. 检查类别是否选错
4. 检查框是否普遍过大或过小
5. 检查一张图多个缺陷时是否漏框
6. 修正标注规则
7. 再扩到每类 30 到 50 张
```

我复核时重点看：

```text
同一类缺陷的框选尺度是否一致
相似缺陷是否被标成不同类别
反光、阴影、纹理是否被误标
长条缺陷是否被框得过松
小缺陷是否漏标
标签文件是否和图片一一对应
classes.txt 顺序是否固定
```

第一次训练后只验收三件事：

```text
明显缺陷能不能框出大致位置
正常图误检是否太多
哪些类别最容易漏检或混淆
```

如果第一版效果可以，再扩充到难标类别，例如不导电、擦花、桔皮和轻微浅色缺陷。

#### 阶段三第二轮：以提升 mAP50 为主

第一轮服务器训练已经跑通，当前先把目标收窄到提升 `mAP50`，暂时不追求 `mAP50-95`。

原因：

```text
mAP50 更关注“有没有大致框到缺陷”
mAP50-95 更关注“框的位置和大小是否非常精细”
当前现场目标是辅助人工发现缺陷，先要让模型能稳定框出大致位置
等大致位置稳定后，再优化框得准不准、紧不紧
```

第一轮服务器训练结果摘要：

```text
训练集/验证集：120 / 30 images
总样本：5 类各 30 张
最终 mAP50：0.503
最佳 mAP50：0.503
```

分类别结论：

```text
crack：mAP50 0.922，已经学得比较好，第二轮少量补难例即可
transverse_bump：mAP50 0.592，有初步效果，继续补形态差异
dent：mAP50 0.427，能学到一部分，但漏检偏多
powder：mAP50 0.369，不稳定，需要补更多清晰凸粉
stain：mAP50 0.073，基本没学会，第二轮最高优先级
```

第二轮标注目标：

```text
目标一：把整体 mAP50 从约 0.50 提升到 0.60 以上
目标二：把 stain mAP50 从约 0.07 提升到 0.30 以上
目标三：把 powder 和 dent mAP50 提升到 0.50 左右
目标四：保持 crack 不明显下降
```

第二轮不把 `mAP50-95` 作为主要目标。它只作为观察项记录，等 mAP50 稳定后再优化。

#### 第二轮补标优先级

按优先级补标，不平均用力：

```text
P0：stain 脏点
P1：powder 凸粉
P1：dent 碰伤
P2：transverse_bump 横条压凸/压凹类
P3：crack 涂层开裂
```

建议新增标注量：

```text
stain：新增 60 到 80 张
powder：新增 40 到 60 张
dent：新增 40 到 60 张
transverse_bump：新增 20 到 40 张
crack：新增 10 到 20 张，只补难例和不同形态
```

第二轮完成后，总量大约控制在：

```text
stain：90 到 110 张
powder：70 到 90 张
dent：70 到 90 张
transverse_bump：50 到 70 张
crack：40 到 50 张
```

这不是为了“每类一样多”，而是为了让模型优先补上当前不会的类别。

#### 第二轮挑图规则

优先挑能帮助 mAP50 上升的图片：

```text
缺陷清楚
人工一眼能看出缺陷在哪里
缺陷边界大致可框
缺陷大小、位置、光照和第一轮不完全一样
同一类中尽量覆盖不同形态
```

暂时不要挑：

```text
特别轻微、肉眼都难判断的图
缺陷和反光混在一起的图
整片纹理异常但没有明确局部区域的图
一张图里多类缺陷混在一起且不好区分的图
已经带框、带文字、截图式图片
```

第二轮先不要新增新类别。`不导电`、`擦花`、`桔皮`、`漏底` 后续单独开第三轮处理，避免把当前 5 类还没学稳时又增加难度。

#### 第二轮各类别标注要点

`stain` 脏点：

```text
只框真实脏点，不框大面积背景
多个分散小脏点可以分别框
如果小点很密集连成一片，可以用一个区域框
反光、阴影、灰度渐变不要当脏点
优先标清楚、颜色或纹理明显不同的脏点
```

`powder` 凸粉：

```text
框住凸起粉点或粉块的可见区域
不要把整条铝型材都框进去
细小凸粉如果肉眼很难看清，先放不确定
优先补不同大小、不同位置、不同背景亮度的凸粉
```

`dent` 碰伤：

```text
框住碰伤、凹坑、擦碰痕迹的主体区域
如果周围有明显拖痕，可以连同拖痕一起框
不要把正常边缘、阴影、反光当碰伤
优先补小碰伤、边缘碰伤、浅色碰伤
```

`transverse_bump` 横条压凸/压凹类：

```text
长条缺陷用长框
框住实际横向异常区域，不要把正常长边也框进去
同一张图有多条横向异常时，可以分多个框
优先补不同宽度、不同深浅、不同位置的横条
```

`crack` 涂层开裂：

```text
当前效果已经好，少量补难例即可
优先补细裂纹、短裂纹、低对比裂纹
不要再大量补和第一轮很像的明显长裂纹
```

#### 第二轮批次安排

推荐从 `batch_004` 开始继续：

```text
batch_004：优先 stain，目标 60 张，已完成 57 张有效标注
batch_005：powder + dent，各 30 到 40 张
batch_006：stain 小批复核 20 到 30 张，只保留非常确定的脏点
batch_007：根据 batch_005 和 batch_006 训练结果，再决定是否继续补 transverse_bump 和 crack 难例
```

`batch_004` 服务器训练结论：

```text
dataset_annotated_207:
  train images: 166
  val images: 41
  stain total: 87

server_train_207:
  final mAP50: 0.405
  best mAP50: 0.450
  stain mAP50: 0.0757
```

这个结果说明：单纯增加脏点数量没有明显提升 `stain`，整体 `mAP50` 也低于上一轮。因此下一步不要继续盲目补大量脏点，应先复核脏点标注规则。

脏点复核重点：

```text
不确定是不是脏点的图片直接跳过，不要为了凑数量强行标
只框真实可见的黑点、污点、小块污染区域
不要把大面积阴影、反光、铝型材纹理变化框进去
一张图多个独立脏点可以画多个小框，不要用一个很大的框包住半张图
如果缺陷太小、边界太模糊，宁可跳过
```

`batch_005` 服务器训练结论：

```text
dataset_annotated_265:
  train images: 212
  val images: 53
  dent total: 58
  powder total: 60
  stain total: 87

server_train_265:
  final mAP50: 0.351
  best mAP50: 0.394
  dent mAP50: 0.354
  powder mAP50: 0.255
  stain mAP50: 0.0683
```

这个结果继续下降，说明当前阶段的主要矛盾已经不是“图片数量不够”，而是下面三件事：

```text
标注框尺度是否一致
同一类别的缺陷定义是否稳定
验证集是否能代表真实想检测的缺陷
```

下一步建议先暂停大批量补标，改做标注质量复盘：

```text
1. 从 batch_004 和 batch_005 各抽 20 张，检查框是否过大、过小、漏框或误框。
2. 对 dent、powder、stain 分别整理 3 到 5 张“标准标注样例图”。
3. 对不确定图片明确标记 skipped，不进入训练。
4. 重新导出一个只包含高质量标注的 clean dataset 做对照训练。
5. 下一次服务器训练使用 batch 64，提高 A6000 显存利用率。
```

每个 batch 都必须有：

```text
images/
labels/
classes.txt
predefined_classes.txt
selection_manifest.csv
```

继续遵守：

```text
已经 annotated 的图片不要重复标注
图片可以保留在本地，但不要保存到 Git
Git 只保存 labels、registry、manifest、脚本和文档
```

#### 第二轮训练验收

每轮标注完成后，重新导出干净数据集并训练。验收只看三个主要指标：

```text
整体 mAP50 是否提升
stain / powder / dent 的 mAP50 是否提升
val_batch0_pred.jpg 中框是否落在真实缺陷附近
```

第二轮达标线：

```text
整体 mAP50 >= 0.60：说明第二轮有效
stain mAP50 >= 0.30：说明脏点开始学会
powder mAP50 >= 0.50：说明凸粉可进入下一轮优化
dent mAP50 >= 0.50：说明碰伤可进入下一轮优化
```

如果整体 mAP50 没提升，优先检查：

```text
新标注框是否过大
类别是否选错
是否混入了太难、太模糊、太轻微的图片
验证集是否太少导致波动
```

## 5. 大缺陷和小缺陷处理

先保证明显缺陷，再逐步处理小缺陷。

第一步优先：

```text
明显碰伤
明显脏点
明显擦花
明显凸粉
涂层开裂
```

第二步再优化：

```text
小脏点
轻微擦花
细小凸粉
浅色缺陷
反光区域缺陷
```

如果小缺陷在整图缩放后看不清，后续需要切图检测：

```text
整图拍摄
把大图切成小块
每个小块单独检测
把检测框合并回原图
```

## 6. 数据与标注建议

供货商图片回来后，不要马上全部加入训练。

建议先做：

```text
图片质量筛选
人工分类
抽样检查
再加入训练集
```

如果要训练检测模型，需要人工标注缺陷框。

第一批标注建议：

```text
每类缺陷 30 到 50 张
优先标注擦花、碰伤、凸粉、脏点
总量先控制在 200 到 500 张
```

标注要求：

```text
框出大致缺陷区域即可
不要追求像素级贴边
一张图多个缺陷就标多个框
看不清的图先放入不确定
坏图不要用于训练
```

推荐标注工具：

```text
LabelImg
CVAT
Label Studio
```

如果只是先做小批量验证，可以先用 LabelImg。后续多人协作再考虑 CVAT 或 Label Studio。

## 7. 推荐实施顺序

第一步：整理人工复核结果。

```text
确认缺陷判正常样本
确认未分类中哪些是真正常图
确认四类重点错分原因
```

第二步：供货商图片回来后做质量筛选。

```text
剔除坏图
剔除重复图
剔除水印、截图、标注图
按 11 类和混合缺陷整理
```

第三步：继续优化分类模型。

```text
补充正常样本
补充弱类样本
减少缺陷判正常
保留 Top1 + Top3 辅助判断
```

第四步：准备检测模型小样本验证。

```text
挑选 200 到 500 张缺陷图
人工框出大致缺陷区域
训练 YOLO 检测模型
查看能否框出大致位置
```

第五步：现场演示版。

```text
相机拍照
模型推理
屏幕显示结果
异常图片保存
人工决定是否拿走
```

## 8. 当前不建议做的事

当前阶段不建议直接做：

```text
PLC 自动剔除
复杂流水线追踪
像素级精细分割
一次性标注大量图片
一开始就购买复杂硬件
```

原因：项目是否落地、图片质量、现场拍摄方式、客户预算还没有完全确定。现在应先用低成本方式验证分类、显示和粗定位是否能满足人工辅助需求。
