# 铝型材表面缺陷检测项目流程 PPT 大纲

## 1. 项目目标

标题建议：铝型材表面缺陷检测与缺陷区域判断

讲解口径：
- 目标不是一次性替代人工，而是先做“缺陷类型识别 + 缺陷区域提示”。
- 当前路线分两层：分类模型判断图片属于哪种缺陷，检测模型进一步框出缺陷位置。
- 后续现场使用时，模型给出提示，人工复核高风险样本。

## 2. 数据与缺陷类别

讲解口径：
- 原始数据已经按标准类别分类，适合先做分类模型。
- 缺陷形态差异很大：有些是明显局部缺陷，例如涂层开裂、碰伤；有些是纹理类缺陷，例如桔皮、擦花、不导电，框选难度更高。
- 因此阶段三优先选择容易框选、边界清楚的类别。

## 3. 阶段一：分类模型

讲解口径：
- 分类模型解决“这张图像大概率属于哪类缺陷”。
- 它适合做第一层筛选：从大量图片里挑出高置信样本，减少人工挑图成本。
- 但分类模型只能告诉类别，不能稳定告诉缺陷在图中哪里。

建议表达：
- 输入：单张铝型材表面图片。
- 输出：缺陷类别和置信度。
- 用途：高置信挑图、辅助后续标注、现场快速预警。

## 4. 阶段二：Grad-CAM 可解释性

讲解口径：
- Grad-CAM 用来观察分类模型主要看图像的哪个区域。
- 它不是精确标注框，只能作为“模型关注区域”的参考。
- 如果 Grad-CAM 热区和真实缺陷位置接近，说明分类模型学到了一部分有效特征。

## 5. 阶段三：人工框选标注

讲解口径：
- 阶段三开始做目标检测，需要人工用 LabelImg 给缺陷画框。
- 本轮先标注边界更清楚的类别：碰伤、凸粉、脏点、涂层开裂、横条压凸/压凹等。
- 不导电、擦花、桔皮这类纹理类缺陷暂缓，因为边界不稳定，容易引入噪声。

建议配图：
- `images/07_annotation_contact_sheet.jpg`

## 6. 标注到训练的闭环

讲解口径：
- 图片由分类模型先筛选高置信样本。
- 人工只标注筛选后的样本，降低工作量。
- 标注完成后记录清单，避免重复标注。
- 只保存标注文件和清单到 Git，原始图片不放入 Git。

建议流程：
1. 分类模型筛选高置信图片。
2. 按类别放入待标注目录。
3. 使用 LabelImg 画框。
4. 导出 YOLO 数据集。
5. 上传服务器训练。
6. 下载结果并复盘类别表现。

讲解口径：

## 10. 下一轮优化方向

讲解口径：
- 当前优先目标是提升 mAP50，不急着追求 mAP50-95。
- mAP50 主要衡量“是否大致框到缺陷”，更符合当前阶段验证目标。
- 下一轮重点补充低分和样本不足类别。

建议计划：
- P0：补脏点 60-80 张，目标 stain mAP50 从 0.073 提升到 0.30 以上。
- P1：补凸粉、碰伤各 40-60 张，目标 mAP50 到 0.50 以上。
- P2：补横条压凸/压凹 20-40 张，继续稳定规则形缺陷。
- P3：涂层开裂作为稳定类别，少量补充难例。

## 11. 现场落地流程

讲解口径：
- 现场不是只看模型分数，而是形成“拍照、推理、提示、复核、留档”的闭环。
- 前期模型只负责提示风险区域，最终判断仍可由人工确认。
- 每次人工确认后的样本可继续进入下一轮训练，模型会逐步改善。

建议表达：
- 相机采图。
- 分类模型判断类别。
- 检测模型框出疑似缺陷。
- 屏幕提示缺陷位置和置信度。
- 人工复核并保存误检、漏检样本。
- 进入下一轮训练。
