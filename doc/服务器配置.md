## 6. YOLO 阶段三服务器训练流程

### 6.1 SSH 连接信息

本项目后续阶段三 YOLO 检测训练默认使用 `xmu-server`。本机已经配置免密登录。

```sshconfig
Host xmu-server
    HostName 100.75.43.2
    User guanjianxiong
    Port 22
```

备用 IPv6 地址保留记录：

```text
fd7a:115c:a1e0::9f01:2b14
```

连通性检查：

```bash
ssh xmu-server "hostname && nvidia-smi --query-gpu=index,name,memory.total,memory.used --format=csv"
```

### 6.2 服务器工作目录

训练文件放到数据盘，避免占用根目录。

```text
/data/guanjianxiong/visualinspection/yolo_stage3
  dataset_annotated_150/
  scripts/
  outputs/
```

本地 Git 只保存标注结果和元数据，不保存标注图片。需要训练时从本地已导出的干净数据集上传：

```text
data/yolo_stage3_manual/exports/dataset_annotated_150
```

当前数据集规模：

```text
train images: 120
val images: 30
classes: dent, powder, stain, crack, transverse_bump
```

### 6.3 上传命令

在本地项目根目录执行：

```powershell
ssh xmu-server "mkdir -p /data/guanjianxiong/visualinspection/yolo_stage3/scripts /data/guanjianxiong/visualinspection/yolo_stage3/outputs"

scp -r data/yolo_stage3_manual/exports/dataset_annotated_150 xmu-server:/data/guanjianxiong/visualinspection/yolo_stage3/

scp scripts/train_yolo_stage3.py xmu-server:/data/guanjianxiong/visualinspection/yolo_stage3/scripts/
```

如果服务器没有 `yolov8n.pt`，训练脚本会让 Ultralytics 自动下载；若服务器不能联网，则需要手动上传本地权重到工作目录。

### 6.4 启动训练

优先使用 GPU 0，输出写入 `/data` 盘：

```bash
ssh xmu-server "cd /data/guanjianxiong/visualinspection/yolo_stage3 && nohup python3 scripts/train_yolo_stage3.py \
  --data dataset_annotated_150/data.yaml \
  --model yolov8n.pt \
  --epochs 100 \
  --imgsz 640 \
  --batch 16 \
  --device 0 \
  --workers 8 \
  --project outputs \
  --name server_train_150 > outputs/server_train_150.log 2>&1 &"
```

### 6.5 10 分钟检查流程

Codex heartbeat 已设置为每 10 分钟检查一次，任务 ID：

```text
yolo-stage3-training-follow-up
```

每次检查执行：

```bash
ssh xmu-server "ps -ef | grep train_yolo_stage3.py | grep -v grep || true"
ssh xmu-server "tail -n 80 /data/guanjianxiong/visualinspection/yolo_stage3/outputs/server_train_150.log || true"
ssh xmu-server "ls -lah /data/guanjianxiong/visualinspection/yolo_stage3/outputs/server_train_150 || true"
```

训练完成后重点查看：


### 6.6 结果回传与有效性判断

训练结束后回传关键结果：

```powershell
mkdir server_outputs\yolo_stage3_server_train_150
scp -r xmu-server:/data/guanjianxiong/visualinspection/yolo_stage3/outputs/server_train_150/training_report.md server_outputs/yolo_stage3_server_train_150/
scp -r xmu-server:/data/guanjianxiong/visualinspection/yolo_stage3/outputs/server_train_150/results.csv server_outputs/yolo_stage3_server_train_150/
scp -r xmu-server:/data/guanjianxiong/visualinspection/yolo_stage3/outputs/server_train_150/results.png server_outputs/yolo_stage3_server_train_150/
scp -r xmu-server:/data/guanjianxiong/visualinspection/yolo_stage3/outputs/server_train_150/val_batch0_pred.jpg server_outputs/yolo_stage3_server_train_150/
```

初步有效性判断标准：

```text
1. 训练能完整跑完，存在 weights/best.pt。
2. results.csv 中 mAP50 明显高于本地 50 epoch CPU 试跑结果。
3. val_batch0_pred.jpg 中框的位置覆盖真实缺陷，而不是只学到背景纹理。
4. 若 precision 高但 recall 低，说明漏检明显，下一批标注应优先补充同类缺陷的形态差异。
5. 若 prediction 图中框偏大或偏移，优先检查 LabelImg 标注框是否过宽或类别是否错位。
```
