# 铝型材缺陷图片分类统计说明书

更新时间：2026-06-16

## 1. 供货商图片提供要求

本项目检测对象是**铝型材**，不是钢材。请供应商提供铝型材现场图片。

请按以下要求准备图片：

1. 请提供正常铝型材图片和不良铝型材图片。
2. 图片不要加红框、红字、箭头、水印或截图标注。
3. 每种常见铝型材形状都请提供一些样本。
4. 尽量固定拍摄角度、距离、光源和背景。
5. 不良图片中缺陷位置要清晰可见，包括擦花、碰伤、漏底、脏点、凸粉、涂层开裂等。
6. 每种铝型材形状建议先提供正常图 800-1000 张、不良图 200-300 张。
7. 如果有现场相机或工位，请尽量用未来实际检测位置拍摄，不要只用手机随手拍。
8. 如果缺陷不明显，请在图片文件名中写出缺陷位置说明，例如“左上”“中间”“右下”“边缘”。
9. 如果一张图中有多种缺陷，请放入混合类型目录，并在文件名中写出组合类型。

## 5. 缺陷图片示例

以下图片来自当前训练数据，仅用于说明供货商应提供的图片类型。供应商不需要在图片上画框或加字，只需要提供原始清晰图片。

### 5.1 正常/合格品

![正常/合格品](说明图片/00_normal_clean_sample.jpg)

说明：表面整体一致，没有明显擦伤、脏点、凸起、开裂等缺陷。供应商需要为每种常见铝型材形状提供足够正常样本。

### 5.2 不导电

![不导电](说明图片/01_non_conducting.jpg)

说明：表面处理异常，可能体现为局部色泽、覆盖或导电性能异常。此类目前样本较少，需要优先补充。

### 5.3 擦花

![擦花](说明图片/02_scuffing.jpg)

说明：表面存在擦伤、刮擦、拉伤类痕迹。此类与正常、脏点、凸粉容易混淆，供应商应提供轻微、中等、明显多种程度。

### 5.4 横条压凹

![横条压凹](说明图片/03_transverse_strip_dented.jpg)

说明：表面存在横向条状压痕或凹陷。供应商应尽量提供不同宽度、不同位置、不同深浅的样本。

### 5.5 桔皮

![桔皮](说明图片/04_orange_peel.jpg)

说明：表面呈现不均匀纹理，类似桔皮状。拍摄时需要保持光源稳定，否则容易和正常纹理混淆。

### 5.6 漏底

![漏底](说明图片/05_drain_bottom.jpg)

说明：表面覆盖不足，底层或颜色差异明显。此类当前样本数量相对较多，但仍建议补充不同形状和不同产线样本。

### 5.7 碰伤

![碰伤](说明图片/06_collision_damage.jpg)

说明：铝型材被撞击、磕碰、挤压后产生局部破损、凹坑或变形。此类当前识别效果一般，需要重点补充。

### 5.8 起坑

![起坑](说明图片/07_pitting.jpg)

说明：表面存在坑点、凹点或局部点状异常。供应商应提供小坑、大坑、密集坑、边缘坑等样本。

### 5.9 凸粉

![凸粉](说明图片/08_convex_powder.jpg)

说明：表面出现凸起粉点、颗粒或小白点。此类和擦花、脏点存在混淆，需要更多清晰样本。

### 5.10 涂层开裂

![涂层开裂](说明图片/09_coating_cracking.jpg)

说明：涂层出现裂纹或断裂。当前样本较少，建议补充不同裂纹长度、宽度、方向的样本。

### 5.11 脏点

![脏点](说明图片/10_dirty_spot.jpg)

说明：表面有黑点、污点、杂质点或污染痕迹。此类容易和碰伤、凸粉混淆，需要提供不同大小和不同颜色深浅的样本。

### 5.12 混合类型

混合类型是指一张图片中同时存在两种或多种缺陷。

示例目录：

```text
图片/混合类型/擦花+脏点/
图片/混合类型/碰伤+漏底/
图片/混合类型/凸粉+脏点/
```

命名示例：

```text
20260616_line01_shapeA_scuffing+dirty_spot_擦花中间_脏点右上_0001.jpg
20260616_line01_shapeB_collision+drain_bottom_碰伤左侧_漏底中间_0002.jpg
```

当前判断：多缺陷图片可能比较少，短期可以通过“图片裁剪后分别计算，再合成一张结果图”的方式处理。

## 8. 当前模型训练方式

当前采用的是图片级单标签分类模型：

```text
整张图片 -> ResNet50 分类模型 -> 输出一个类别 + 置信度
```

当前模型能做：

- 判断整张图属于哪个缺陷类别。
- 输出预测类别和置信度。
- 导出 TorchScript，供后续调用。

当前模型不能做：

- 不能准确框出缺陷位置。
- 不能可靠识别一张图中的多个缺陷。
- 不能输出每个缺陷的坐标范围。

服务器上关键模型文件：

```text
/home/guanjianxiong/code/yolo/VisualInspection/outputs/classification_results/deploy/classifier.torchscript.pt
/home/guanjianxiong/code/yolo/VisualInspection/outputs/classification_results/deploy/labels.json
/home/guanjianxiong/code/yolo/VisualInspection/outputs/classification_results/models/model_best.pth.tar
```

备份优先级：

1. `classifier.torchscript.pt`：部署调用模型，必须备份。
2. `labels.json`：类别映射文件，必须和模型一起备份。
3. `model_best.pth.tar`：最佳训练权重，建议备份。

## 8.1 模型优化记录与对比思路

当前优化目标不是机械执行固定流程，而是提高铝型材 11 类分类准确度，尤其降低“有缺陷但被判成正常”的漏检风险。

已完成记录：

- 当前基线模型：ResNet50 图片级单标签分类。
- 部署文件：`outputs/classification_results/deploy/classifier.torchscript.pt`。
- 标签文件：`outputs/classification_results/deploy/labels.json`。
- 训练权重：`outputs/classification_results/models/model_best.pth.tar`。
- 全量已分类图片筛选结果：Top1 准确率 92.45%，Top3 命中率 99.37%。
- 未分类 `raw_images` 筛选结果：1271 张图片处理成功，442 张被模型判断为正常候选，0 张坏图。

当前弱项类别：

| 类别 | 当前问题 | 优化方向 |
| --- | --- | --- |
| 擦花 | 容易和正常、脏点、凸粉混淆 | 增加样本，优先用类别权重和难样本训练 |
| 碰伤 | 容易和脏点混淆 | 增加不同形状、不同严重程度样本 |
| 凸粉 | 容易和擦花、脏点混淆 | 增加清晰小颗粒样本 |
| 脏点 | 容易和正常、碰伤、擦花混淆 | 增加不同大小、颜色深浅样本 |

后续服务器实验策略：

| 实验 | 运行参数 | 目的 |
| --- | --- | --- |
| Class Weight | `--class-weights` | 提高少样本类别权重 |
| Weighted Sampler | `--weighted-sampler` | 训练时增加少样本类别出现频率 |
| Focal Loss | `--loss focal --focal-gamma 2.0` | 让模型更关注难分样本 |
| Class Weight + Focal Loss | `--class-weights --loss focal --focal-gamma 2.0` | 防漏检组合实验 |

对比时不要只看总准确率。推荐按下面顺序判断：

1. 先看“缺陷判正常”数量是否减少。
2. 再看擦花、碰伤、凸粉、脏点四类是否提升。
3. 再看 Top1 总准确率是否下降过多。
4. 最后看正常图片误报是否可以接受。

如果某个实验 Top1 总准确率略低，但漏检明显减少，可以作为候选模型保留。因为当前系统定位是人工辅助，漏检比误报更重要。

## 11. 后续最优路线

### 11.1 第一阶段：先把 11 类粗分类做稳

当前类别先固定 11 类，不先扩展新类别。

目标：

```text
输入图片 -> 输出 top1/top3 类别 + 置信度 + 是否需要人工复核
```

训练重点：

- 重点补充擦花、碰伤、脏点、凸粉。
- 对错分图片进行人工复核，清理标错样本。
- 增加 top3 输出，不只看 top1。
- 增加低置信度策略，例如低于 0.70 自动进入人工复核。

漏检更重要，因此策略应偏保守：

- 只要怀疑有缺陷，就不要轻易判为正常。
- 正常类阈值可以设置得更严格。
- 低置信度图片进入人工复核。

### 11.2 第二阶段：裁剪后分类，处理少量多缺陷情况

多种缺陷同时出现的可能性目前判断较小。短期可以采用裁剪策略：

```text
原图 -> 切成多个区域 -> 每个区域分类 -> 合成整图结果
```

优点：

- 不需要马上训练检测框模型。
- 可以提高局部小缺陷的发现概率。
- 最终可以把多个区域结果合成到一张图上显示。

缺点：

- 计算量增加。
- 缺陷框位置只是粗略区域，不是真正精确框。
- 裁剪大小需要根据现场图片分辨率调参。

### 11.3 第三阶段：训练框选模型

如果要真正框出受损位置，最终还是建议训练检测模型。

推荐：

- YOLO：用于缺陷框检测。
- ONNX：作为主要导出格式，方便后续调用。
- OpenVINO：如果工控机使用 Intel CPU，可以用于加速。
- TensorRT：如果工控机上 NVIDIA GPU，可以用于加速。

目标输出：

```json
{
  "has_defect": true,
  "defects": [
    {
      "class_name": "擦花",
      "confidence": 0.86,
      "box": [120, 80, 260, 130]
    }
  ]
}
```

如果供应商不能提供框标注，可以先询问是否能提供；如果不能提供，就需要内部人工标注少量样本，先从擦花、碰伤、脏点、凸粉开始。

## 12. 工控机部署考虑

当前工控机型号还未确定，原则是尽量节省成本。

初步要求：

- 目标速度：先按 1 秒 10 张图片考虑，后续可调节。
- CPU：优先考虑 Intel CPU。
- GPU：如果 CPU 达不到速度要求，可以上 NVIDIA GPU。
- 部署格式：优先导出 ONNX，方便后续系统调用。
- 图片来源：最好固定现场相机、固定工位、固定光源、固定距离。

建议推理路线：

```text
现场图片
  -> 预处理
  -> 分类模型 top3
  -> 必要时裁剪区域再次分类
  -> 后续检测模型框选
  -> 合成结果图
  -> 人工复核
```

性能优化方向：

- 控制输入图片尺寸。
- 分类模型可对比 ResNet50、ResNet18、MobileNet、EfficientNet。
- CPU 部署优先测试 ONNX Runtime。
- Intel CPU 可测试 OpenVINO。
- NVIDIA GPU 可测试 TensorRT。
- 只对疑似缺陷区域做二次计算，减少全图重复计算。

## 13. 后续可选能力：严重程度判断

后续如果计算量和数据都允许，可以增加缺陷严重程度判断。

例如：

```text
轻微
中等
严重
```

但这需要供应商或人工给出严重程度标准，否则模型无法稳定学习。

建议等 11 类分类稳定、框选能力稳定后，再做严重程度判断。
