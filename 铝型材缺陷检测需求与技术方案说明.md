# 铝型材表面缺陷检测：数据标注要求、参考代码与云服务器部署指南

本文档记录了本项目的缺陷数据标注规范、参考代码文件清单，以及将模型与代码部署到 GPU 云服务器运行与监控的实操指南。

---

## 1. 🏷️ 数据标注与缺陷分类规范 (Labeling Requirements)

为了确保缺陷分类模型与定位模型能够高精度运行，数据集采用以下标注与整理规范：

### 1.1 核心分类标注映射 (11大类)
主数据集目录位于 `data/ali2018/`。所有样本根据其文件夹名称进行绝对分类标注，对应类别 ID 映射如下：
*   `Clean sample` -> 类别 `0` (正常/合格品)
*   `non-conducting` -> 类别 `1` (不导电)
*   `scuffing` -> 类别 `2` (擦花)
*   `The transverse strip is dented` -> 类别 `3` (横条压凹)
*   `Orange peel` -> 类别 `4` (桔皮)
*   `Drain bottom` -> 类别 `5` (漏底)
*   `Be injured by a collision` -> 类别 `6` (碰伤)
*   `pitting` -> 类别 `7` (起坑)
*   `Convex powder` -> 类别 `8` (凸粉)
*   `Coating cracking` -> 类别 `9` (涂层开裂)
*   `Dirty spot` -> 类别 `10` (脏点)

### 1.2 稀缺小样本隔离标注
*   **隔离策略**：对于 17 类、共 148 张小样本/稀有缺陷图像，统一隔离存放于 `新增少样本缺陷/` 子目录下，不参与主分类器训练，防止类别极度不均衡破坏模型收敛。

### 1.3 测试盲盒图片标记
*   **标记策略**：500 张纯数字测试集图片（`0.jpg`~`499.jpg`）统一重命名为带 `test_` 前缀的标记文件（如 `test_000.jpg`），存放在 `测试文件_未标注/` 中。

### 1.4 物理缺陷框选（Bounding Box）画框规则
*   **定位权重**：使用专门在铝型材数据集上训练好的 YOLO 定位权重（`best.pt`）。
*   **框上标签标注**：自动读取文件所属的缺陷类别（如所在文件夹名字 `scuffing`），在画好的红色缺陷框上方自动打印对应的中文标记（如 `擦花`），将框选结果输出到 `outputs/visualizations/`。

## 🖥️ 3. 云服务器上云部署与运行监控指南 (Cloud Deployment & Monitoring)

本节记录了将代码与数据部署至 `server2` 云服务器，并独占单张显卡运行与监控的实操步骤。

### 3.1 云端目录布局
服务器代码目录与本地项目目录保持同级结构。图片数据在服务器和本地都放到项目目录下的 `data/ali2018/`。

云端目录结构如下：
```text
~/code/yolo/VisualInspection/
├── data/
│   └── ali2018/                  <--- 原始分类图片数据集 (包含 Clean sample 等大类目录)
├── src/
```

路径和训练参数已经写入 `src/classification/config.py`：

* Windows 本地数据：`data/ali2018`
* Linux 服务器数据：`data/ali2018`
* GPU：`CUDA_VISIBLE_DEVICES=0`
* 模型：`resnet50`
* 输入尺寸：`384`
* batch size：`96`
* Linux workers：`16`
* DataLoader prefetch factor：`4`
* CUDA AMP 混合精度：开启
* 训练 epoch：`30`

### 3.2 显卡独占配置
为了不影响服务器上其他显卡的使用，在服务器配置字典（`config.py` 中的 `LINUX_CONFIG`）里，指定只使用 **GPU 0（单张显卡）**。系统在运行时会自动应用此环境变量：
```python
# 限制 PyTorch 仅能检测到 GPU 0
os.environ["CUDA_VISIBLE_DEVICES"] = "0"
```

### 3.3 云端后台挂起运行 (nohup)
为防止因 SSH 终端网络断开导致训练异常中止，必须在后台挂起运行，并将终端日志输出重定向：
```bash
nohup python src/classification/run_pipeline.py > train.log 2>&1 &
```

正常情况下不需要再传 `--env`、`--data-root`、`--epochs`、`--batch-size` 等参数；脚本会自动识别 Windows/Linux 并读取 `config.py` 中的默认配置。

RTX A6000 单卡 48GB 显存下，默认 batch size 已调整为 96。若 `nvidia-smi` 长时间显示显存仍明显低于 40GB 且 GPU-Util 偏低，可临时试：
```bash
python src/classification/run_pipeline.py --batch-size 128
```

若出现 CUDA out of memory，则改回：
```bash
python src/classification/run_pipeline.py --batch-size 96
```

若 AMP 在服务器环境中报兼容问题，可临时关闭：
```bash
python src/classification/run_pipeline.py --no-amp
```

本地或服务器首次验证流程时，可先用小样本 smoke run 检查代码链路，输出到独立目录，避免覆盖正式模型：
```bash
python src/classification/run_pipeline.py \
  --epochs 1 \
  --architecture resnet18 \
  --batch-size 2 \
  --workers 0 \
  --image-size 64 \
  --max-samples 22 \
  --output-dir outputs/classification_results/smoke
```

训练完成后，流水线会同时生成两类模型文件：

* `outputs/classification_results/models/model_best.pth.tar`：训练检查点，用于继续训练、复现或再次导出。
* `outputs/classification_results/deploy/classifier.torchscript.pt`：工控机调用用的 TorchScript 部署模型。
* `outputs/classification_results/deploy/labels.json`：类别 ID、中文名称、提交标签与输入尺寸说明。

如果只需要把已有检查点重新导出为工控机调用模型，可执行：
```bash
python src/classification/export_model.py \
  --checkpoint outputs/classification_results/models/model_best.pth.tar \
  --output-dir outputs/classification_results/deploy
```

工控机或上位机可通过命令行进程调用单张图片推理，输出 JSON：
```bash
python src/classification/predict_single.py \
  --image /path/to/image.jpg \
  --model outputs/classification_results/deploy/classifier.torchscript.pt
```

批量图片可输出 CSV：
```bash
python src/classification/predict_batch.py \
  --input /path/to/images \
  --output outputs/classification_results/predictions.csv \
  --model outputs/classification_results/deploy/classifier.torchscript.pt
```

### 3.5 模型抽样评估与图片结果保存

训练完成后，可运行抽样评估脚本，自动完成两步检查：

1. 从 11 个已标注大类中随机抽取每类最多 30 张图片，不够 30 张则全取，按真实类别分文件夹保存。
2. 从 `测试文件_未标注/` 中随机抽取 200 张，按模型预测类别分文件夹保存。

默认读取：

* 分类部署模型：`outputs/classification_results/deploy/classifier.torchscript.pt`
* 图片数据：`data/ali2018`
* 输出目录：`outputs/classification_results/evaluation_samples`
* YOLO 定位权重：`best.pt`，如果不存在则只绘制分类结果文字，不画缺陷框。

一键运行：
```bash
python src/classification/evaluate_model.py
```

输出结构：
```text
outputs/classification_results/evaluation_samples/
├── labeled_by_truth/             # 已标注图片，按真实类别分文件夹
├── unlabeled_by_prediction/      # 未标注测试图片，按预测类别分文件夹
├── labeled_report.csv            # 已标注抽样预测结果与准确率明细
└── unlabeled_report.csv          # 未标注抽样预测明细
```

如需强制不使用 YOLO，只看分类效果：
```bash
python src/classification/evaluate_model.py --no-yolo
```

### 3.4 云端进程与运行监控指令
在云端运行时，使用以下指令监控系统状态：

*   **实时查看训练日志输出**：
    ```bash
    tail -f train.log
    ```
*   **查看系统训练进度与精度日志**：
    ```bash
    tail -f outputs/classification_results/resnet50_training_log.txt
    ```
*   **查看 Python 进程及 CPU 子线程（验证是否正在运行）**：
    ```bash
    ps -ef | grep run_pipeline.py
    ```
    *(会输出 1 个主进程和 16 个负责数据加载的 CPU 工作子进程)*
*   **滚动刷新查看 GPU 0 单卡占用及显存变化**：
    ```bash
    nvidia-smi -l 1
    ```
