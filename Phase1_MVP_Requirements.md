# 钢材/铝型材表面缺陷检测 - 第一版 (MVP) 需求与环境配置说明书

## 1. 阶段目标概述
本项目第一阶段（MVP版）的核心目标是实现**“离线图片文件夹批量检测工具”**。
暂不接入相机与自动化流水线硬件，专注于验证视觉算法在现有缺陷图片（破损、擦伤、夹渣、滴脏、划痕等）上的**检出率**和**误判率**。系统接收图片文件夹，自动输出带标注框的图片以及结构化的检测结果表。

---

## 2. 本地数据现状分析 (基于 `d:/code/VisualInspection/` 目录)

经过对本地文件夹的初步扫视，当前数据情况如下：

### 2.1 数据源组成
*   **不良图片示例** 文件夹：共计 88 张图片。
    *   格式包含 `.jpg` 和 `.png`。
    *   命名规则多样：包含哈希值命名（如 `04912919e46b...jpg`）、截图命名（如 `ScreenShot_2026-06-05_...png`）、以及具体缺陷命名（如 `拉伤.png`, `拉伤2.png`）。
*   **铝型材缺陷图** 文件夹及压缩包（约 1.1GB）：有大量潜在的可用训练/测试数据。

### 2.2 数据格式与标注要求
*   **清洗要求**：特别需要注意 `ScreenShot` 截图系列图片。如果这些图片上**已经带有系统生成的红框或文字**，**绝对不能**直接投入模型训练，否则模型会学习“如何寻找红框”而不是“如何寻找缺陷”。
*   **统一格式**：算法在读取时需兼容 `.jpg`, `.png`, `.bmp` 等常见工业相机输出格式。
*   **标注输出**：第一版统一输出为 YOLO 格式的 `.txt` 标注文件。

## 3. 开发与运行环境要求

为了保证开发与后续在 Windows 工控机上的顺利部署，第一版采用以下技术栈：

### 3.1 硬件要求 (开发与初期测试)
*   **操作系统**：Windows 10/11 64位
*   **GPU (推荐)**：NVIDIA 显卡（显存 4GB 以上），需配置 CUDA 环境以加速训练和推理。如果仅做推理测试，CPU 也可运行，但速度较慢。

### 3.2 软件与依赖库 (Python 栈)
*   **Python 版本**：Python 3.9 或 3.10（兼容性最佳）。
*   **核心深度学习框架**：`torch`, `torchvision` (匹配本地 CUDA 版本)。
*   **视觉算法核心**：`ultralytics` (YOLOv8 / YOLO11 官方库)，用于目标检测。
*   **图像处理与辅助**：`opencv-python` (图像读取、画框预处理), `numpy`, `pandas` (生成结果表格)。

---

## 4. 第一版 (MVP) 核心功能设计

### 4.1 输入模块 (Input)
*   支持通过命令行或简单配置文件，指定**“待检测图片文件夹路径”**（例如 `d:/code/VisualInspection/待测试图片/`）。
*   支持指定 YOLO 权重文件路径（例如 `best.pt`）。

### 4.2 推理模块 (Inference)
*   自动遍历输入文件夹内的所有图片。
*   对图片进行缩放、归一化预处理。
*   送入 YOLO 模型，提取疑似缺陷的坐标 (Bounding Box)、类别 (Class) 和 置信度分数 (Confidence)。

### 4.3 输出模块 (Output)
程序运行结束后，需在指定的“输出文件夹”中生成以下内容：
1.  **可视化结果图**：在原图上画出矩形框，并在框的上方标注缺陷类别和置信度，供人工复核。
2.  **检测结果汇总表 (`result.csv` 或 `result.xlsx`)**：
    包含以下字段，方便追溯和统计：
    *   `图片名称`
    *   `判定结果` (OK / NG)
    *   `缺陷数量`
    *   `缺陷详情` (类别、置信度、坐标 x_min, y_min, x_max, y_max)

## 5. 下一步行动项 (Action Items)

为了将上述设计转化为实际可运行的代码，建议按以下步骤执行：

*   [ ] **Action 1：清理并准备训练数据**
    从 `不良图片示例` 和 `铝型材缺陷图` 中，挑选出**没有任何红框/文字污染的纯净原图**，建立 `dataset/images/train` 文件夹。
*   [ ] **Action 2：执行人工标注**
    使用 AnyLabeling 或 LabelImg 对纯净原图进行框选，保存为 YOLO 格式。
*   [ ] **Action 3：编写并运行 Python 训练代码**
    编写 `train.py` 脚本，基于少量的标注数据（如 100 张）训练一个基线模型（Baseline Model）。
*   [ ] **Action 4：编写并运行批量检测脚本**
    编写 `batch_detect.py`，实现上述的输入、推理与输出表格功能。
